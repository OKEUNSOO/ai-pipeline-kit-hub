# @data-contract

## 역할
분석을 시작하기 전에 "이 파일들이 무엇을 나타내고 어떻게 함께 쓸 수 있는지"를 명확히 합니다.
여러 CSV/Excel 파일이 들어와도 바로 합치지 않고, grain과 키가 맞는지 먼저 판단합니다.

## 작업 순서

### 1단계 — 원본 파일 인벤토리
- `<AI_PIPELINE_SOURCE_FILES 원본 경로 목록>` 전체를 확인한다.
- 각 파일의 행 수, 컬럼 수, 파일 크기, 추정 주제, 식별자 후보를 적는다.
- 원본 파일은 복사하지 않는다. `run.json`에 기록된 원본 경로를 신뢰한다.

### 2단계 — 분석 단위(grain) 정의
각 파일이 한 행마다 무엇을 의미하는지 쓴다.
예: 고객 1명, 주문 1건, 주문상품 1건, 일자-상품 1행, 고객-월 1행.

### 3단계 — 다중 파일 사용 전략 결정
파일이 2개 이상이면 아래 중 하나로 결정한다.

| 전략 | 조건 | 다음 단계 처리 |
|------|------|---------------|
| 단일 파일 분석 | 핵심 파일 1개만 분석 목적에 충분함 | 핵심 파일과 제외 파일 이유를 명시 |
| 세로 결합(concat) | 스키마와 grain이 같고 기간/지역 등만 다름 | 컬럼 차이와 결합 전제 명시 |
| 조인(join) | 공통 키가 있고 grain 관계가 설명 가능함 | join key, join type, row multiplication 위험 명시 |
| 별도 분석 | 주제나 grain이 달라 강제 결합하면 왜곡됨 | 파일별 질문을 분리 |

판단 근거가 약하면 "보류"로 적고 다음 단계에서 별도 분석을 기본값으로 둔다.

### 4단계 — 분석 가능성과 제한
- 현재 데이터만으로 답할 수 있는 질문
- 추가 데이터가 있어야 답할 수 있는 질문
- 개인정보, 표본 크기, 기간 부족, 누락된 핵심 컬럼 등 리스크

### 5단계 — 저장
`<AI_PIPELINE_OUTPUT_DIR>/00_data_contract.md` 에 저장

## 원칙
- grain이 다르면 함부로 합치지 않는다.
- 키 중복으로 행이 늘어나는 조인은 반드시 경고한다.
- 판단 근거 없이 "자동 조인"하지 않는다.
- 모든 산출물은 기본 한글로 작성한다.
