# 분석 품질 원칙

모든 에이전트에 공통 적용됩니다.

- **KPI 정의가 먼저** — 이름, 계산 기준, 단위를 확정한 뒤 분석
- **절대값 / 상대값 / 누적값 혼용 금지**
- **차트는 질문에 답한다** — 예쁨보다 명확함 우선
- **카드 제목은 결론형 또는 질문형** — "매출 현황"이 아니라 "왜 마진이 낮은가?"
- **임원용 요약과 실무자용 상세는 분리**
- **표본이 작으면 반드시 경고**
- **산출물은 기본 한글 작성** — 보고서와 대시보드의 사용자-facing 문구는 한글로 쓰되, 원본 컬럼명·파일명·계산식·코드 식별자는 정확성을 위해 유지
- **다중 파일은 계약 먼저** — 여러 파일은 `00_data_contract.md`에서 grain, 키, concat/join/분리 전략을 정한 뒤 분석
- **지표는 기계가 읽을 수 있어야 한다** — 핵심 KPI는 `04_kpi_summary.md`뿐 아니라 유효한 `04_metrics.json`에도 저장
- **재현 가능한 분석** — 보고서와 대시보드 핵심 수치는 `05_analysis.py`로 다시 계산할 수 있고, `05_analysis.ipynb`에서 분석 흐름을 검토할 수 있어야 한다
- **최종 검증 필수** — 제출 전 `08_validation_report.md`에서 수치 일관성, 재현성, 해석 리스크를 확인
- **도메인 독립성** — 커머스 전용 가정을 타 도메인에 강제하지 않는다
